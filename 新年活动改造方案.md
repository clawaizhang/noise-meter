# 新年活动改造方案

## 背景分析

### 现有7天签到活动概况
- **活动名称**: 新年特别活动
- **活动周期**: 7天连续签到
- **核心奖励**: 保底180天会员，最高可得1095天会员（3年）
- **每日基础奖励**: 第1-6天25天，第7天30天
- **随机加成**: 0-130天随机奖励，按概率分布
- **用户参与度**: 每人限参与一次，活动有效期为30天

### 当前代码结构
1. **BenefitActivityDialog.ets** - 主要活动弹窗组件
2. **MembershipRewardService.ets** - 奖励计算和签到逻辑
3. **BenefitActivityData.ets** - 活动数据模型
4. **PreferenceKeys.ets** - 偏好设置存储
5. **MembershipStatusCard.ets** - 活动入口展示

## 产品设计目标

### 主要目标
1. **主题转换**: 从通用"7天签到"转换为"新年特别活动"
2. **成本优化**: 减少会员赠送天数，控制运营成本
3. **提升体验**: 增强节日氛围和用户参与感
4. **时间限定**: 增加节日时效性，促进及时参与

### 设计原则
- **节日氛围**: 突出新年主题，使用春节相关视觉元素
- **价值感知**: 虽减少总天数，但通过设计和包装提升价值感
- **用户友好**: 保持简单易懂的参与规则
- **技术可行**: 基于现有代码结构最小化修改

## 新年活动方案详情

### 1. 活动基本信息
- **活动名称**: "龙年大吉，签到有礼"
- **活动主题**: 春节/新年特别活动
- **活动周期**: 7天（保持原周期，但可考虑缩短为5天）
- **活动时间**: 限定在新年前后（如除夕至初七）
- **参与条件**: 所有用户均可参与，每人限一次

### 2. 视觉主题设计
- **主色调**: 红色、金色（春节传统色彩）
- **图标元素**: 🐉 龙、🧧红包、🎆烟花、🏮灯笼
- **字体风格**: 喜庆、传统书法风格元素
- **背景图案**: 春节装饰图案（春联、福字元素）

### 3. 奖励结构调整（减少会员天数）

#### 方案A：温和调整（推荐）
| 签到天数 | 原基础奖励 | 新基础奖励 | 减少幅度 |
|---------|-----------|-----------|---------|
| 第1天 | 25天 | 15天 | -40% |
| 第2天 | 25天 | 15天 | -40% |
| 第3天 | 25天 | 20天 | -20% |
| 第4天 | 25天 | 20天 | -20% |
| 第5天 | 25天 | 25天 | 0% |
| 第6天 | 25天 | 25天 | 0% |
| 第7天 | 30天 | 30天 | 0% |

**随机加成调整**:
- 原范围: 0-130天
- 新范围: 0-60天（减少约54%）
- 概率分布向低值倾斜

**保底奖励调整**:
- 原保底: 180天会员
- 新保底: 120天会员（减少33%）
- 最高可得: 约500天（原1095天）

#### 方案B：显著调整（成本控制优先）
| 签到天数 | 新基础奖励 | 累计奖励 |
|---------|-----------|---------|
| 第1天 | 10天 | 10天 |
| 第2天 | 10天 | 20天 |
| 第3天 | 15天 | 35天 |
| 第4天 | 15天 | 50天 |
| 第5天 | 20天 | 70天 |
| 第6天 | 20天 | 90天 |
| 第7天 | 30天 | 120天 |

保底奖励: 90天会员
随机加成: 0-30天

### 4. 特殊节日奖励机制
- **除夕特别奖励**: 第1天签到额外+10天
- **初一开门红**: 第2天签到额外+15天
- **连续签到加成**: 3天、5天、7天有额外节日红包（小额外奖励）
- **完成奖励**: 7天完成赠送春节限定主题或图标

### 5. 活动时间范围
- **启动时间**: 除夕前3天（预热期）
- **核心活动期**: 除夕至初七（8天窗口，覆盖7天签到）
- **结束时间**: 元宵节当天
- **用户参与窗口**: 活动期间内任意连续7天

### 6. 用户参与逻辑
1. **资格检查**: 活动期间内首次打开应用自动获得资格
2. **签到中断**: 允许中断后继续累计（原逻辑是中断重置）
3. **完成状态**: 完成7天签到后标记为已完成
4. **奖励应用**: 实时应用到会员有效期
5. **活动提示**: 首页显著位置展示活动入口和倒计时

## 产品价值分析

### 成本效益
- **总奖励减少**: 约40-50%的会员天数赠送
- **运营成本**: 显著降低长期会员负债
- **用户感知**: 通过节日包装提升价值感，抵消天数减少的影响

### 用户体验提升
- **节日氛围**: 增强应用的新年气氛
- **参与动机**: 时间限定增加紧迫感和参与意愿
- **情感连接**: 春节主题增强用户情感共鸣

### 商业目标达成
- **用户活跃度**: 春节期间提升DAU/MAU
- **留存率**: 通过7天签到培养使用习惯
- **品牌形象**: 展示节日关怀，提升品牌好感度

## 风险与应对

### 技术风险
1. **数据兼容性**: 新老用户数据冲突
   - 应对: 添加版本标记，区分新老活动数据

2. **奖励计算错误**: 可能导致用户投诉
   - 应对: 充分测试，添加日志记录，准备应急修复方案

### 用户风险
1. **用户不满**: 相比原活动奖励减少
   - 应对: 强调节日特殊性、限定性，包装为"新年特别福利"
   
2. **参与度下降**: 奖励减少导致签到动力不足
   - 应对: 增加节日元素和完成成就感，如成就徽章

### 运营风险
1. **时间窗口错过**: 用户未能在活动期间完成
   - 应对: 适当延长活动时间窗口，添加提醒功能
