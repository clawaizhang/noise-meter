# 历史记录页面菜单按钮重构总结

## 任务目标
将历史记录页面右上角的两个独立按钮（收藏、编辑）改造为一个菜单按钮，点击后弹出包含"收藏"、"删除记录"、"删除音频"三个选项的菜单。

## 实现方案

### 1. 状态管理重构
- 将原有的 `shouChangMode` 和 `deleteMode` 状态分离为三个独立状态：
  - `shouChangMode`: 收藏模式
  - `deleteRecordMode`: 删除记录模式  
  - `deleteAudioMode`: 删除音频模式
- 添加 `showMenu` 状态控制菜单显示

### 2. 菜单按钮实现
- 使用 `bindContextMenu` 方法实现菜单功能
- 配置菜单显示位置为 `Placement.BottomRight`
- 使用三点菜单图标作为菜单按钮

### 3. 菜单构建器 (MenuBuilder)
- 创建自定义 `MenuBuilder` 构建器
- 包含三个菜单项：
  - 收藏（带收藏图标）
  - 删除记录（带删除图标）
  - 删除音频（带音频分析图标）
- 每个菜单项都有独立的点击处理逻辑

### 4. 功能逻辑
- 点击菜单项后自动关闭菜单
- 各模式之间互斥（开启一个模式时自动关闭其他模式）
- 保持原有的收藏和删除功能逻辑不变

## 技术要点

### 使用的组件和方法
- `bindContextMenu`: 绑定上下文菜单
- `@Builder`: 自定义构建器装饰器
- `Placement.BottomRight`: 菜单位置配置
- 状态管理：使用 `@Local` 装饰器

### 解决的问题
1. **UI简化**: 将两个按钮合并为一个菜单按钮，界面更简洁
2. **功能分离**: 将编辑功能细分为"删除记录"和"删除音频"两个独立选项
3. **用户体验**: 提供更直观的菜单交互方式
4. **错误修复**: 修复了 `stateStyles` 使用导致的运行时错误
