# 瀑布图调试检查清单

## 数据流检查

### 1. 数据源验证
- [ ] 确认音频处理服务正在生成频谱数据
- [ ] 检查FFT分析器是否正常工作
- [ ] 验证频谱数据格式是否正确（Float32Array）

### 2. WaterfallData数据流
- [ ] `addSpectrumFrame`方法是否被调用
- [ ] 频谱数据是否通过`isValidSpectrum`验证
- [ ] 数据压缩和动态分辨率处理是否正常
- [ ] `spectrumHistory`数组是否在增长

### 3. 响应式数据更新
- [ ] `@Monitor`装饰器是否触发数据变化监听
- [ ] `displayWaterfallData`是否在UI状态中正确更新
- [ ] 数据变化是否触发组件重绘

## Canvas和渲染器检查

### 4. Canvas上下文验证
- [ ] Canvas组件是否正确初始化
- [ ] Canvas尺寸是否设置正确
- [ ] Canvas上下文是否可用
- [ ] 清除画布操作是否执行

### 5. ImageData渲染器检查
- [ ] ImageData渲染器是否成功初始化
- [ ] ImageData对象是否创建成功
- [ ] 数据缓冲区是否可用
- [ ] 像素设置操作是否执行

### 6. 像素绘制流程
- [ ] 处理后的帧数据是否包含有效bin
- [ ] 颜色值计算是否正确
- [ ] 像素坐标是否在画布范围内
- [ ] 像素数据是否添加到缓冲区
- [ ] `flush()`操作是否执行

## 并行计算检查

### 7. 颜色映射生成
- [ ] 颜色映射参数是否正确传递
- [ ] 并行任务是否成功执行
- [ ] 颜色映射数组是否生成
- [ ] 颜色值格式是否正确（RGBA字符串）

### 8. 帧数据处理
- [ ] 并行帧处理任务是否成功
- [ ] 处理后的帧是否包含有效数据
- [ ] bin数据是否包含有效的db值和颜色
- [ ] 坐标计算是否正确

## 颜色映射检查

### 9. 颜色配置
- [ ] 颜色映射类型设置是否正确
- [ ] 最小/最大分贝值设置是否合理
- [ ] 透明度设置是否生效
- [ ] 颜色值计算是否在有效范围内

### 10. 颜色计算
- [ ] 归一化计算是否正确
- [ ] 颜色索引计算是否准确
- [ ] RGB值是否在0-255范围内
- [ ] 透明度值是否在0-1范围内

## 性能问题检查

### 11. 绘制频率
- [ ] 绘制间隔是否合理（50ms限制）
- [ ] 是否因为频繁重绘导致闪烁
- [ ] 数据更新频率是否过高

### 12. 内存使用
- [ ] 内存池管理是否正常
- [ ] 是否有内存泄漏
- [ ] 数据缓存是否有效

## 错误处理检查

### 13. 异常捕获
- [ ] 所有异步操作是否有错误处理
- [ ] 并行计算失败是否有回退机制
- [ ] 错误信息是否详细记录

## 调试步骤

### 第一步：基础数据验证
1. 检查控制台是否有数据相关的日志输出
2. 确认`spectrumHistory`数组长度是否大于0
3. 验证第一帧数据的有效性和范围

### 第二步：绘制流程验证
1. 确认`drawWaterfall`函数被调用
2. 检查Canvas上下文是否可用
3. 验证ImageData渲染器初始化状态

### 第三步：并行计算验证
1. 检查颜色映射生成是否成功
2. 验证帧数据处理是否返回有效结果
3. 确认处理后的帧数据格式正确

### 第四步：像素绘制验证
1. 检查像素数据是否生成
2. 验证像素坐标计算
3. 确认`flush()`操作执行

### 第五步：性能优化
1. 检查绘制耗时
2. 优化数据更新频率
3. 验证内存使用情况

## 常见问题解决方案

### 问题1：白色屏幕
- **可能原因**: 数据为空或绘制失败
- **解决方案**: 添加数据验证和错误处理日志

### 问题2：闪烁
- **可能原因**: 频繁重绘或数据更新过快
- **解决方案**: 调整绘制间隔，优化数据更新频率

### 问题3：颜色异常
- **可能原因**: 颜色映射计算错误
- **解决方案**: 验证颜色参数和计算逻辑

### 问题4：性能问题
- **可能原因**: 计算量过大或内存泄漏
- **解决方案**: 优化算法，添加内存监控
