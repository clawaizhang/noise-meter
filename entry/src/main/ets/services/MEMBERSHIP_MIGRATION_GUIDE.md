# 遗留会员迁移 - AGC后台配置指南

## 概述
本文档描述如何将原有"连续使用送会员"的本地会员数据迁移到华为IAP系统。

## 迁移策略

### 1. 遗留会员判定
- 检查本地存储的 `local_expire_time`
- 如果未过期且未转换过 (`legacy_converted=false`)，则为遗留会员

### 2. 迁移方案

#### 方案A: 永久会员 (遗留天数 > 1095天，即3年)
- **目标商品**: `vip_permanent` (非消耗型)
- **价格设置**: ¥0 (或¥0.01如果系统不支持¥0)
- **用户获得**: 终身会员资格

#### 方案B: 订阅会员 (遗留天数 ≤ 1095天)
- **目标商品**: `vip_monthly` (自动续期订阅)
- **促销优惠**: 根据遗留天数赠送对应月数
- **用户获得**: 订阅会员，促销期结束后自动续费

## AGC后台配置步骤

### 步骤1: 创建永久会员商品 (NONCONSUMABLE)

1. 登录 [AppGallery Connect](https://developer.huawei.com/consumer/cn/service/josp/agc/index.html)
2. 进入您的项目 → 应用内支付 → 商品管理
3. 点击"添加商品"
   - **商品ID**: `vip_permanent`
   - **商品类型**: 非消耗型
   - **价格**: ¥0 (或¥0.01)
   - **名称**: 永久会员
   - **描述**: 终身免费使用所有功能，换设备登录自动恢复

### 步骤2: 创建订阅商品 (AUTORENEWABLE)

1. 点击"添加商品"
   - **商品ID**: `vip_monthly`
   - **商品类型**: 自动续期订阅
   - **订阅周期**: 1个月
   - **价格**: ¥6/月
   - **名称**: 月度会员
   - **描述**: 解锁全部专业功能

### 步骤3: 配置促销优惠 (关键步骤)

为 `vip_monthly` 配置5种促销优惠，对应不同的遗留会员天数：

| 遗留天数 | 赠送月数 | 优惠ID | 促销类型 | 优惠时长 | 优惠价格 |
|---------|---------|--------|---------|---------|---------|
| 1-89天 | 3个月 | `offer_legacy_3m` | 推介促销 | 3个月 | ¥0.01 |
| 90-179天 | 6个月 | `offer_legacy_6m` | 推介促销 | 6个月 | ¥0.01 |
| 180-364天 | 12个月 | `offer_legacy_1y` | 推介促销 | 12个月 | ¥0.01 |
| 365-729天 | 24个月 | `offer_legacy_2y` | 推介促销 | 24个月 | ¥0.01 |
| 730-1095天 | 36个月 | `offer_legacy_3y` | 推介促销 | 36个月 | ¥0.01 |

**配置方法**:
1. 进入 `vip_monthly` 商品详情
2. 点击"添加促销优惠"
3. 选择"推介促销" (Introductory Offer)
4. 设置优惠时长和价格
5. 记录优惠ID（用于代码中）

### 步骤4: 配置普通试用优惠 (可选)

为新用户配置7天免费试用：
- **优惠ID**: `offer_trial_7d`
- **类型**: 推介促销
- **时长**: 7天
- **价格**: ¥0

## 代码实现要点

```typescript

const legacy = await vipService.hasLegacyVoucher();

// 2. 执行转换
if (legacy.days > 1095) {
  // 创建永久会员购买 (¥0)
  await vipService.convertLegacyVoucher();
} else {
  // 创建订阅购买 (带促销优惠)
  await vipService.convertLegacyVoucher();
}

// 3. 处理购买结果
// - 解析JWS
// - 发放权益
// - finishPurchase确认
// - 标记本地已转换
```

### 关键API调用
```typescript
// 订阅转换（带促销优惠）
const result = await iap.createPurchase(context, {
  productId: 'vip_monthly',
  productType: iap.ProductType.AUTORENEWABLE,
  promotionalOfferId: 'offer_legacy_3m', // 【关键】传入优惠ID
  developerPayload: JSON.stringify({
    type: 'legacy_convert',
    originalDays: 60,

  })

```

## 注意事项

### 1. 价格设置问题
- 华为IAP最低价格为 ¥0.01
- 设置 ¥0.01 并在商品描述中说明这是"象征性费用，实际为免费转换"

### 2. 促销优惠限制
- 每个订阅商品最多配置1个推介促销
- 用户只能享受1次推介促销
- 确保遗留会员转换的促销配置正确

### 3. 自动续费说明（重点）
- 订阅会员转换后，促销期结束会自动续费（¥6/月）
- **必须在多处UI上明确告知用户：**
  1. **遗留会员转换页面**: 显示红色提示条 "免费期结束后自动按¥6/月续费，可随时取消"
  2. **转换确认弹窗**: 明确告知用户促销期后的续费规则
  3. **订阅会员卡片**: 显示"自动续费已开启/关闭"状态
  4. **非会员试用页面**: 提示"试用到期前24小时自动续费"
  5. **服务协议弹窗**: 第2条明确列出自动续费条款
- 提供"管理订阅"入口让用户可以随时取消自动续费

### 4. 测试验证
在正式发布前，必须完成以下测试：

1. **沙盒测试**
   - 使用华为测试账号
   - 验证 ¥0.01 购买是否成功
   - 验证促销优惠是否正确应用

2. **遗留会员场景测试**
   - 模拟不同天数的遗留会员
   - 验证正确的促销优惠被应用
   - 验证转换后权益正确发放

3. **异常处理测试**
   - 网络中断恢复
   - 购买过程中取消
   - 重复转换尝试

## 常见问题

### Q: 用户转换后是否可以退款？
A: 转换价格为 ¥0.01（象征性费用），金额极小，一般不建议用户申请退款。如果用户坚持，可以通过华为IAP的退款流程处理。

### Q: 如果用户已经有订阅，还能转换吗？
A: 不能。代码中已经做了检查 (`canPurchaseProduct`)，如果已有生效订阅，需要先取消才能转换。

### Q: 转换失败怎么办？
A: 保留本地会员数据，显示错误信息，允许用户重试。本地会员可以继续使用直到过期。

### Q: 如何统计转换成功率？
A: 建议在 `developerPayload` 中标记转换类型，并在服务器端记录转换日志。
