# 马年抽签活动方案

## 方案核心创意

### 活动机制变更
- **原机制**: 固定天数奖励 + 随机加成
- **新机制**: 每次签到抽取"吉祥签"，不同签文对应不同奖励天数
- **周期**: 5天连续签到
- **总奖励**: 大幅降低，采用吉祥数字

### 设计理念
1. **节日氛围**: 抽签形式增强春节传统文化体验
2. **趣味性**: 每次签到都有惊喜感
3. **成本控制**: 总体奖励天数大幅减少
4. **吉祥寓意**: 结合马年主题设计吉祥签文

### 5天签到奖励计算
**期望值计算**:
- 单次签到期望奖励 ≈ 14.5天
- 5天总期望奖励 ≈ 72.5天
- **保底总奖励**（全部抽到最低）: 10天
- **最高总奖励**（全部抽到最高）: 140天

### 抽签算法
```typescript
/**
 * 执行抽签
 * @returns 抽中的吉祥签
 */
private drawLuckySign(): LuckySign {
  const random = Math.random();
  let cumulative = 0;
  
  for (const sign of LUCKY_SIGNS) {
    cumulative += sign.probability;
    if (random <= cumulative) {
      return sign;
    }
  }
  
  // 保底：返回最低奖励签
  return LUCKY_SIGNS[LUCKY_SIGNS.length - 1];
}
```

/**
 * 执行福利活动签到（抽签版）
 * @returns 签到结果
 */
public signBenefitActivity(): BenefitSignResult {
  // ... 原有检查逻辑（活动有效、未签到等）
  
  // 执行抽签
  const luckySign = this.drawLuckySign();
  
  // 更新连续天数
  benefitData.currentStreak++;
  
  // 记录签到日期和奖励
  benefitData.signDates.push(Date.now());
  benefitData.lastSignDate = Date.now();
  
  // 记录本次抽签结果
  benefitData.todayLuckySign = luckySign.id;
  benefitData.todayReward = luckySign.rewardDays;
  
  // 更新累计奖励
  benefitData.totalRewardDays += luckySign.rewardDays;
  
  // 检查是否完成5天签到
  if (benefitData.currentStreak >= 5) {
    benefitData.benefitActivityCompleted = true;
  }
  
  // 应用奖励
  this.applyBenefitReward(luckySign.rewardDays);
  
  return {
    success: true,
    day: benefitData.currentStreak,
    baseReward: luckySign.rewardDays, // 抽签奖励作为baseReward
    bonusReward: 0, // 不再有额外加成
    totalRewardDays: benefitData.totalRewardDays,
    isCompleted: benefitData.benefitActivityCompleted,
    luckySign: luckySign // 新增字段
  };
}

## 推荐方案

### 推荐采用**完整抽签方案**（10种签文）
**理由**:
1. **节日氛围强**: 抽签形式与传统春节文化高度契合
2. **用户体验好**: 每次签到都有惊喜感和期待感
3. **成本控制佳**: 期望奖励72.5天，相比原180天减少60%
4. **品牌价值**: 展现文化底蕴和创新精神
5. **差异化**: 与市面上普通签到活动形成差异化

### 风险控制
1. **技术风险**: 抽签动画可能影响性能，需要优化
2. **用户期望**: 部分用户可能抽到低奖励签不满意
3. **数据统计**: 需要跟踪各签文的抽取概率和用户反馈
