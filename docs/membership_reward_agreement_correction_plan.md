# 会员奖励计划协议修正计划

## 问题描述
会员奖励计划协议中的连续使用奖励描述与实际实现不一致。

## 当前协议描述（不准确）
在 [`MembershipAgreementDialog.ets`](entry/src/main/ets/components/membership/MembershipAgreementDialog.ets:52-58) 中：
```
1. 每日使用奖励：每天使用APP达到1分钟，即可获得1天会员资格
2. 连续使用奖励：
   • 连续使用3天，额外获得2天会员奖励
   • 连续使用7天，额外获得3天会员奖励
   • 连续使用15天，额外获得5天会员奖励
3. 奖励累计：所有奖励天数会自动累加到您的会员有效期
```

## 修正方案
更新协议描述以准确反映实际奖励规则：

### 新的协议描述内容
```
1. 每日使用奖励：每天使用APP达到1分钟，即可获得1天会员资格
2. 连续使用奖励：
   • 连续使用1天：获得1天会员
   • 连续使用2天：获得2天会员
   • 连续使用3天：获得5天会员
   • 连续使用4天：获得3天会员
   • 连续使用5天：获得4天会员
   • 连续使用6天：获得5天会员
   • 连续使用7天：获得10天会员
   • 连续使用8天：获得5天会员
   • 连续使用9天：获得6天会员
   • 连续使用10天：获得15天会员
   • 连续使用15天：获得30天会员
   • 连续使用30天：获得60天会员
3. 奖励累计：所有奖励天数会自动累加到您的会员有效期
4. 基础规则：不在里程碑中的天数，按连续天数获得对应天数会员
```

## 备注
- 实际实现中，不在里程碑中的天数会使用基础规则：连续天数 = 赠送天数，但不超过最近里程碑的奖励
- 奖励在使用完成时（达到60秒使用时长）才发放
- 连续天数中断后，将从第一天重新开始计算
