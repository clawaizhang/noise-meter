## 视觉设计细节

### 警告区域样式
```
警告区域:
- 背景色: #FFF3E0 (浅橙色背景)
- 边框: 1px solid #FFB74D
- 图标: ⚠️ 橙色警告图标
- 文字: #E65100 (深橙色)
- 按钮: 主要按钮样式，橙色主题
```

### 状态颜色规范
- **正常状态**: 绿色 (#00BC7E)
- **警告状态**: 橙色 (#FF9500) 
- **开关开启**: 品牌色 (#007AFF)
- **开关关闭**: 灰色 (#C7C7CC)

## 交互流程演示

### 用户操作流程
```
1. 用户打开应用，进入噪音警报页面
2. 系统检测到通知权限被拒绝
3. 显示警告信息："系统通知已关闭，请开启系统通知"
4. 用户点击"去设置"按钮
5. 跳转到系统通知设置页面
6. 用户开启通知权限
7. 返回应用，界面自动更新为正常状态
8. 显示："警报触发时，系统将发送通知"
```

### 权限状态变化
```
权限状态变化:
正常 → 拒绝: 立即显示警告
拒绝 → 正常: 立即隐藏警告，显示正常描述
```

## 用户体验优势

### 1. 清晰的状态指示
- 用户一眼就能看出权限状态
- 颜色和图标提供直观的视觉反馈

### 2. 便捷的操作路径
- 直接提供"去设置"按钮
- 减少用户操作步骤

### 3. 一致的体验
- 两个位置的提示信息完全一致
- 相同的视觉风格和交互方式

### 4. 及时的反馈
- 权限状态变化时立即更新界面
- 用户获得即时确认

## 技术实现说明

### 权限检查时机
- 组件加载时检查
- 界面可见时检查
- 定时轮询检查（可选）

### 状态管理
```typescript
interface NotificationState {
  appEnabled: boolean;      // 应用内开关状态
  systemPermission: boolean; // 系统权限状态
  showWarning: boolean;     // 是否显示警告
}
```

### 界面更新策略
- 使用响应式状态管理
- 条件渲染警告区域
- 平滑的过渡动画

这个设计方案将有效提升用户在系统通知权限被拒绝时的体验，提供清晰的指引和便捷的操作路径。
