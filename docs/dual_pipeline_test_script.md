# 双管线架构测试脚本

## 测试目的
验证双管线架构重构后，APP_INPUT_BLOCK错误是否得到解决，前后台切换时UI更新是否正确控制。

## 测试步骤

### 1. 编译和安装
```bash
# 在DevEco Studio中编译应用
# 安装到测试设备
```

### 2. 基础功能测试
1. 启动应用
2. 验证分贝测量功能正常
3. 验证频谱显示功能正常
4. 验证统计数据更新正常

### 3. 前后台切换测试
1. 应用在前台运行，观察UI更新频率
2. 切换到后台（按Home键或切换到其他应用）
3. 观察日志输出，确认看到"[双管线架构] UI更新已暂停"
4. 等待30秒，确认后台无UI刷新事件积压
5. 切换回前台
6. 观察日志输出，确认看到"[双管线架构] UI更新已恢复"
7. 验证UI数据正确同步

### 4. 性能监控测试
1. 使用DevEco Studio性能分析器监控
2. 检查vSync事件队列长度
3. 检查主线程阻塞情况
4. 检查内存使用情况

### 5. 错误监控测试
1. 检查系统日志中APP_INPUT_BLOCK错误
2. 监控应用稳定性
3. 验证无崩溃或异常退出

## 预期结果

### 成功标准
- [ ] 前后台切换时正确输出日志信息
- [ ] 后台运行时无UI刷新事件积压
- [ ] 前台恢复时UI数据正确同步
- [ ] APP_INPUT_BLOCK错误为0
- [ ] 应用运行稳定，无崩溃

## 问题排查

### 常见问题
1. **UI状态不同步**
   - 检查AudioController中的syncFromProcessingState调用
   - 验证UIDisplayState的isUIUpdatesEnabled状态

2. **后台仍有UI更新**
   - 检查EntryAbility中的pauseUIUpdates调用
   - 验证AudioControllerService实例设置

3. **数据统计不连续**
   - 检查DataProcessingState的统计更新逻辑
   - 验证StatisticsData的updateStatistics方法

### 调试方法
1. 添加详细日志输出
2. 使用DevEco Studio调试器
3. 检查状态变量值
