## 改进的提示信息设计

### 更清晰的提示文字
**标题**: "系统通知权限提醒"
**描述**: 
```
系统通知权限被拒绝
即使开启应用通知，也无法收到警报
```

**按钮**: "去系统设置开启通知"

### 设计思路
1. **明确问题**: "系统通知权限被拒绝" - 直接说明问题
2. **解释影响**: "即使开启应用通知，也无法收到警报" - 说明后果
3. **明确操作**: "去系统设置开启通知" - 清晰的操作指引

### 对比改进
- **之前**: "系统通知已关闭，请开启系统通知以接收警报"
- **现在**: "系统通知权限被拒绝，即使开启应用通知，也无法收到警报"

## 用户理解流程

### 用户看到的信息层次
1. **第一层**: 应用设置正常（开关开启，描述正常）
2. **第二层**: 权限问题警告（独立卡片，醒目提示）
3. **第三层**: 解决方案（明确的按钮操作）

### 用户认知过程
```
用户看到: "警报触发时，系统将发送通知" 
    ↓
同时看到: "系统通知权限被拒绝，即使开启应用通知，也无法收到警报"
    ↓
理解: 应用设置没问题，但系统权限有问题
    ↓
操作: 点击"去系统设置开启通知"
```

## 技术实现要点

### 显示逻辑
```typescript
// 显示警告卡片的条件
const showPermissionWarning = 
  appNotificationEnabled &&    // 应用内开关开启
  !systemPermissionGranted;    // 系统权限被拒绝

// 应用内描述文字保持不变
const appDescription = appNotificationEnabled ? 
  '警报触发时，系统将发送通知' : 
  '通知已关闭，系统不会通知噪音超标情况';
```

### 用户引导
- 按钮点击后跳转到系统通知设置页面
- 提供清晰的权限开启指引
- 返回应用后自动刷新权限状态

这个设计确保了用户能够清晰理解问题所在，并提供明确的解决方案。
