## 独立警告区域设计

### 警告卡片样式
```
┌─────────────────────────────────────────┐
│  ⚠️ 系统通知权限提醒                    │
│                                         │
│  系统通知已关闭                         │
│  请开启系统通知以接收警报               │
│                                         │
│  [去设置]                               │
└─────────────────────────────────────────┘
```

### 视觉设计规范
- **背景色**: #FFF3E0 (浅橙色背景)
- **边框**: 1px solid #FFB74D
- **图标**: ⚠️ 橙色警告图标 (24x24)
- **标题**: #E65100, 字体加粗
- **描述文字**: #E65100, 正常字体
- **按钮**: 主要按钮样式，橙色主题

### 布局结构
```
Column({ space: 12 }) {
  // 标题行
  Row({ space: 8 }) {
    Image($r('app.media.ic_warning'))  // 警告图标
    Text('系统通知权限提醒')            // 标题
  }
  
  // 描述文字
  Text('系统通知已关闭\n请开启系统通知以接收警报')
    .textAlign(TextAlign.Center)
  
  // 操作按钮
  Button('去设置')
    .width('60%')
}
.padding(16)
.background('#FFF3E0')
.border({ width: 1, color: '#FFB74D' })
.borderRadius(12)
```

## 设计优势

### 1. 信息分离
- 应用设置信息保持不变
- 权限警告作为独立卡片显示
- 用户清楚区分两种信息

### 2. 视觉层次
- 警告卡片使用醒目的橙色主题
- 与应用设置区域明显区分
- 重要信息突出显示

### 3. 操作明确
- 独立的"去设置"按钮
- 清晰的指引文字
- 用户知道需要做什么

### 4. 状态清晰
- 权限正常时：不显示警告卡片
- 权限被拒绝时：显示警告卡片
- 状态变化时：动态显示/隐藏

## 交互逻辑

### 显示条件
```typescript
// 当且仅当以下条件满足时显示警告卡片
const showPermissionWarning = 
  systemNotificationEnabled &&  // 应用内开关开启
  !systemPermissionGranted;     // 系统权限被拒绝
```

### 用户流程
```
1. 用户打开应用
2. 系统检查通知权限状态
3. 如果权限被拒绝且应用开关开启 → 显示警告卡片
4. 用户点击"去设置" → 跳转到系统设置
5. 用户开启权限后返回 → 警告卡片自动消失
```

### 状态管理
```typescript
interface PermissionState {
  appSwitchEnabled: boolean;     // 应用内开关状态
  systemPermission: boolean;     // 系统权限状态
  shouldShowWarning: boolean;    // 是否显示警告
}
```

这个独立警告区域的设计确保了应用设置信息的完整性，同时为用户提供了清晰的权限问题解决方案。
